# Тестовое задание для программистов ИБС
## Пояснения к заданию 1.
Схема базы данных представлена в файле "Диаграмма_БД.png", кастомные типы данных, используемые в таблицах, представлены в файле Типы_данных.png.

Таблицы: 
1. Faculty – таблица для хранения факультетов ВУЗа.
2.	Department – таблица для хранения кафедр на факультете.
3.	Specialization – таблица для хранения специальностей, которым могут обучать на кафедре.
4.	Curriculum – учебный план, по которому обучается специальность.
5.	Discipline – таблица, для хранения всех дисциплин.
6.	Discipline_Curriculum – таблица для осуществления связи многие-ко-многим между дисциплинами и учебными планами, так как конкретный учебный план содержит множество дисциплин, а дисциплина может относиться ко множеству учебных планов.
7.	Teacher – таблица для хранения преподавателей.
8.	Teacher_Discipline – таблица для осуществления связи многие-ко-многим между преподавателями и дисциплинами. Необходима для хранения дисциплин, разрешенных для ведения занятий конкретному преподавателю.
9.	Course – таблица для хранения конкретных сущностей дисциплин в конкретном году.
10.	Teacher_Course – таблица для привязки преподавателя к конкретному курсу в конкретном году.
11.	AcademicYear – таблица для хранения академического года (например: «2025/2026»).
12.	Group – таблица для хранения группы студентов обучающихся по конкретному учебному плану.
13.	Group_Course – таблица для осуществления связи многие-ко-многим между группами и курсами. Необходима для хранения конкретных курсов к которым привязываются группы студентов.
14.	Classroom – таблица для хранения аудиторий с типом и вмещаемостью.
15.	ScheduleSlot – таблица для хранения временных интервалов по которым могут проводиться занятия.
16.	Lesson – таблица для хранения конкретных пар в определенное время у определенной группы с определенными преподавателем и в конкретную неделю (четную/нечетную).
17.	Lesson_Group – таблица для осуществления связи многие-ко-многим между парами и группами. Необходима для потоковых занятий, так как на одной паре могут присутствовать много групп.

Типы данных: 
1. DayType – возможные дни учебы (понедельник, …, суббота).
2. TimeSlotType – возможные временные промежутки для пары.
3. ActivityType – тип занятия (лекция, практика, лабораторная).
4. ClassroomType – тип аудитории (лекционная, компьютерный класс, лаборатория, класс для семинаров).
5. WeekType – тип недели (четная, нечетная).

Возможные индексы:

|Таблица|Поле| Цель                 |
|-------|----|----------------------|
|Teacher_Discipline|teacher_id| Все дисциплины преподавателя |
|Teacher_Discipline|discipline_id| Всех преподаватели по дисциплине |
|Teacher_Course|teacher_id| Найти все курсы преподавателя |
|Lesson|teacher_course_id| Найти все занятия преподавателя |
|Group_Course|group_id| Найти все курсы группы |
|LessonGroup|lesson_id| Найти все группы, посещающие занятие |
|LessonGroup|group_id| Найти все уроки группы |

Уникальные:

|Таблица|Поле| Цель |
|-------|----|------|
|Teacher_Discipline|(teacher_id, discipline_id)|Исключение дублирования дисциплин, которые может читать преподаватель|
|Group_Course|(group_id, course_id)|Группа не может быть прикреплена на курс больше, чем 1 раз|
|Lesson_Group|(lesson_id, group_id)|Исключение дублирования конкретного занятия для группы|
|Lesson|(teacher_course_id, scheduleslot_id, week_type)|Исключить ситуацию, когда преподаватель проводит две пары одновременно|
|Lesson|(classroom_id, scheduleslot_id, week_type)|Исключить дублирование пар в одной аудитории в одно и то же время|

Композитные индексы:

|Таблица|Поле| Цель |
|-------|----|------|
|Lesson|(scheduleslot_id, week_type)|Проверять занятость интервала времени по типу недели|
|Lesson|(classroom_id, scheduleslot_id, week_type)|Проверка занятости аудитории в конкретный слот|
|Lesson|(teacher_course_id, scheduleslot_id, week_type)|Проверка занятости преподавателя в конкретный слот|

## Пояснения к заданию 2.

Для ее ручного тестирования нужен только этот файл соответственно.

1. Склонировать репозиторий
2. В консоли соответствующей директории прописать следующие команды:
   1. ```
      python -m venv venv
      ```
   2. ```
      .\venv\Scripts\activate
      ```
   3. ```
      pip install -r .\requirements.txt
      ```
   4. ```
      pytest -v
      ```
